# 全站图片占位与样式评估报告

## 首页 (index.html) 图片需求评估

### 1. Hero 区域
**当前状态**：无图片，纯文字 + CTA

**是否需要图片**：✅ **建议添加**
- **理由**：Hero 是首页最重要的视觉锚点，适当的图片可以增强品牌识别和视觉吸引力
- **呈现方式**：**背景图片（subtle）**
  - 全宽背景，低透明度覆盖层
  - 不干扰文字可读性
  - 或：右侧大图，左侧文字（分栏布局）
- **尺寸建议**：1920x800px (16:6.67) 或 1920x1080px (16:9)
- **样式类**：`.hero-image` 或 `.hero-bg-image`

### 2. 内容精选 (highlights)
**当前状态**：✅ 已有图片占位（highlight-thumb）

**是否需要图片**：✅ **需要保留**
- **理由**：内容精选卡片需要视觉吸引力，图片是必要的
- **呈现方式**：**卡片缩略图（当前已实现）**
  - 横向滚动卡片中的顶部图片
  - 固定宽高比，保持视觉一致性
- **尺寸建议**：320x240px (4:3) 或 320x180px (16:9)
- **样式类**：`.highlight-thumb` ✅ 已存在

### 3. 课程入口 (courses-entry)
**当前状态**：无图片，纯文字描述

**是否需要图片**：⚠️ **可选**
- **理由**：课程入口是功能性导航，图片不是必需的
- **呈现方式**（如果添加）：**小缩略图（左侧）**
  - 每个课程系列左侧小图
  - 尺寸较小，不抢夺文字焦点
  - 或：完全省略，保持纯文本
- **尺寸建议**：120x90px (4:3) 或省略
- **样式类**：`.course-series-thumb`（已在 global-visual-phase3.css 中定义）

### 4. 洞察入口 (insights-entry)
**当前状态**：无图片，纯文字 + 链接

**是否需要图片**：❌ **不建议**
- **理由**：洞察入口是功能性说明文字，添加图片会分散注意力
- **呈现方式**：无

## 图片占位样式系统建议

### 统一占位类命名
1. `.hero-image` - Hero 区域大图
2. `.highlight-thumb` - 内容精选缩略图 ✅
3. `.course-series-thumb` - 课程系列缩略图 ✅
4. `.course-series-cover` - 课程系列封面图（新）
5. `.course-thumb` - 课程卡片缩略图 ✅
6. `.event-thumb` - 活动缩略图 ✅
7. `.insight-thumb` - 洞察缩略图 ✅（但洞察页不使用）

### 统一样式原则
- **宽高比**：统一使用 4:3 或 16:9
- **对象适配**：`object-fit: cover` 保持视觉一致性
- **降权处理**：非主要图片使用低饱和度和透明度
- **响应式**：确保移动端良好显示
