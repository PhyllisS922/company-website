# 课程页面图片需求与结构提示评估

## 当前页面结构

### 标题层级
1. **H1.accent-title**：`课程概览` ✅ 已有橙色竖线
2. **H2.section-title**：`课程系列` ❌ 无橙色竖线
3. **.section-header h2**：`柔新经济特区 30 讲`、`新马营商通识 3 讲` ✅ 已有橙色竖线
4. **H2.section-title**：`课程使用方式` ❌ 无橙色竖线
5. **H2**：`准入与后续步骤` ❌ 无橙色竖线

---

## 图片需求评估

### 1. 课程系列模块（.section-header）
**是否需要图片**：⚠️ **可选，但建议预留**

**理由**：
- 课程系列是核心内容，封面图可以增强视觉识别
- 但课程页面是信息密集型，图片不是必需的
- 如果添加，应该保持克制

**建议呈现方式**：
- **选项A：系列封面图（顶部）** - 推荐
  - 每个课程系列顶部大图
  - 尺寸：800x450px (16:9)
  - 样式类：`.course-series-cover`
  - 位置：在 `.section-header` 之前

- **选项B：小缩略图（左侧）**
  - 左侧小图，右侧文字
  - 尺寸：200x150px (4:3)
  - 样式类：`.course-series-thumb`
  - 位置：在 `.section-header` 内，标题左侧

- **选项C：保持现状（无图片）**
  - 纯文本，更专注内容
  - 保持信息密度

**推荐**：选项A（系列封面图），但保持可选

### 2. 横向滚动卡片（.course-scroll-item）
**是否需要图片**：❌ **不建议**

**理由**：
- 这些是快速浏览的卡片，添加图片会增加视觉噪音
- 当前纯文本更符合"工具性"定位

### 3. 课程使用方式模块
**是否需要图片**：❌ **不需要**

**理由**：
- 这是功能性说明模块，图片会分散注意力

---

## 结构提示增强方案

### 方案1：为 H2.section-title 添加橙色竖线（推荐）

**适用标题**：
- `课程系列`
- `课程使用方式`

**优点**：
- 与 H1 和 .section-header h2 保持一致
- 增强页面结构层级感
- 不会形成视觉噪音（仅用于一级模块标题）

**实现方式**：
```css
body.courses h2.section-title {
    position: relative;
    padding-left: 20px !important;
}

body.courses h2.section-title::before {
    content: "";
    position: absolute;
    left: 0;
    top: 0.2em;
    width: 3px;
    height: 1em;
    background-color: var(--brand-solid);
    opacity: 0.7;
}
```

### 方案2：保持现状，仅 H1 和 .section-header h2 有橙色竖线

**优点**：
- 更克制，避免过度使用
- 突出核心课程系列

**缺点**：
- 一级模块标题（课程系列、课程使用方式）缺少结构提示

### 方案3：为所有 H2 添加橙色竖线

**不推荐**：
- 会形成视觉噪音
- `准入与后续步骤` 是次要信息，不需要强调

---

## 最终建议

### 图片预留
1. **预留课程系列封面图占位**（可选）
   - 使用 `.course-series-cover` 类
   - 在 `.section-header` 之前添加
   - 如果不需要图片，可以隐藏或省略

2. **不添加其他图片**
   - 保持课程页面的信息密度
   - 避免视觉干扰

### 结构提示
**推荐方案1**：为 H2.section-title 添加橙色竖线
- 增强一级模块的结构提示
- 与整体设计系统保持一致
- 不会过度使用（仅2个标题）

---

## 实现优先级

1. **高优先级**：为 H2.section-title 添加橙色竖线
2. **中优先级**：预留课程系列封面图占位（HTML + CSS）
3. **低优先级**：实际添加图片（待内容准备）
